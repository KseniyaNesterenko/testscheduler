# Test Scheduler System

## 1. Назначение системы

Система предназначена для распределения автоматизированных тестов по контейнерам в CI/CD-пайплайне с целью сокращения общего времени выполнения тестов.

Распределение выполняется на основе исторических данных о времени выполнения тестов и применении стратегий планирования.

---

## 2. Основной функционал

- распределение тестов по контейнерам (LPT и расширяемые стратегии)
- сбор и обработка результатов выполнения тестов (JUnit XML)
- хранение статистики времени выполнения тестов
- генерация конфигурации запуска тестов (JSON/YAML)
- REST API для интеграции с CI/CD-системами

---

## 3. Архитектура системы

Система реализована как серверное приложение на базе Spring Boot и включает следующие модули:

- модуль планирования тестов (scheduler)
- модуль обработки отчетов выполнения (collector)
- модуль хранения истории выполнения
- REST API слой для взаимодействия с CI/CD

---

## 4. Требования к окружению

- Java 17
- Docker
- PostgreSQL 16
- Ubuntu 24+ (или аналогичная Linux-система)
